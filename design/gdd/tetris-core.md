# 테트리스 코어 (Tetris Core)

> **Status**: Designed
> **Author**: Luna/Solar Eclipse + Claude
> **Last Updated**: 2026-03-27
> **Implements Pillar**: 매 스테이지가 새로운 퍼즐 (코어 엔진), 2분 안에 결판 (게임 템포 제어)

## Overview

테트리스 코어는 블록 원정대의 핵심 게임플레이 엔진이다. 7종의 테트리미노(I, O, T, S, Z, J, L)를 무작위 순서로 스폰하고, 플레이어의 입력 시그널에 따라 이동·회전·낙하를 처리하며, 보드에 블록을 고정하고 라인 클리어를 실행한다. SRS(Super Rotation System) 기반의 Wall Kick으로 벽 근처에서도 자연스러운 회전을 지원하며, Hold 기능과 Next 프리뷰(1개)로 전략적 선택의 여지를 제공한다. 미션 조건 시스템·스킬 시스템·게임 HUD가 이 시스템의 출력에 의존하며, 테트리스 코어 없이는 어떤 게임플레이도 성립하지 않는다.

## Player Fantasy

테트리스 코어는 플레이어가 의식하지 않아야 성공한 시스템이다. 블록이 손처럼 반응하고, 연속 이동이 자연스럽게 미끄러지며, 회전이 의도한 대로 맞아떨어질 때 — 플레이어는 시스템이 아니라 다음 배치 전략을 생각하고 있어야 한다. "이 블록을 어디에 둘까"라는 판단에 집중할 수 있게 하는 무결한 응답성이 핵심이다. 라인이 클리어될 때의 짧은 쾌감, 하드 드롭이 꽂힐 때의 확정감, 4줄 동시 클리어(테트리스!)의 통쾌함 — 이 감각적 피드백들이 플레이어를 Flow 상태로 유도한다. Pillar 3("2분 안에 결판")의 짧은 긴장감도 이 몰입감에서 온다.

## Detailed Design

### Core Rules

1. **활성 피스 (Active Piece)**
   - 낙하 중인 피스는 `{ type: int, col: int, row: int, rotation: int(0~3) }`로 관리
   - 보드에 기록하지 않음 — 테트리스 코어가 소유
   - `rotation` 상태: 0=기본, 1=CW 90°, 2=180°, 3=CCW 90°

2. **7-Bag 랜덤마이저**
   - 7종(I, O, T, S, Z, J, L)을 한 세트로 셔플 후 순서대로 사용
   - 현재 백이 소진되면 새 백을 즉시 생성
   - Next 프리뷰: 다음 백의 첫 번째 피스도 미리 참조 가능

3. **스폰**
   - 스폰 위치: col=3, row=-1 (스폰 버퍼 최하단, 10열 기준 중앙)
   - I피스 예외: col=3, row=-2 (4칸 너비 고려)
   - 스폰 시 보드와 충돌하면 즉시 GameOver 상태로 전환

4. **이동**
   - `move_left` / `move_right` 시그널 수신 시 col ±1
   - 이동 전 `is_empty()` 체크 — 벽 또는 기존 블록에 막히면 이동 안 함

5. **회전 (SRS)**
   - `rotate_cw` / `rotate_ccw` 시그널 수신 시 rotation 상태 전환
   - 회전 후 충돌 시 SRS Wall Kick 오프셋 테이블(최대 4회 시도) 순서대로 적용
   - 모든 오프셋 실패 시 회전 취소 (원래 상태 유지)

6. **낙하 (타이머 기반)**
   - `fall_timer += delta` 매 `_process()` 프레임 누적
   - `fall_timer >= fall_interval` 시 1칸 낙하 시도, `fall_timer = 0` 리셋
   - 낙하 불가(바닥/블록) 시 Locking 상태 진입

7. **소프트 드롭**
   - `soft_drop_started` 수신 → `effective_fall_interval = base_fall_interval / SDF`
   - `soft_drop_ended` 수신 → `effective_fall_interval = base_fall_interval`

8. **하드 드롭**
   - `hard_drop` 수신 → Ghost 위치(충돌 직전 최하단)까지 즉시 이동 후 고정
   - Lock Delay 없이 즉시 피스 고정

9. **Lock Delay**
   - 피스가 더 이상 낙하 불가 상태가 되면 lock_timer 시작
   - `lock_timer >= LOCK_DELAY` 시 피스 고정
   - Lock Delay 중 이동 또는 회전 성공 시 lock_timer 리셋 (MVP: 최대 리셋 횟수 제한 없음)

10. **피스 고정 및 라인 클리어**
    - 고정 좌표 각 셀에 `place_block()` 호출
    - `get_full_rows()` 호출 → 꽉 찬 행 목록 획득
    - 각 행에 `clear_row()` 호출 (복수 행 동시 클리어 가능)
    - `line_cleared(count: int)` 시그널 발신
    - Hold 쿨다운 해제
    - 새 피스 스폰

11. **Hold**
    - `hold_requested` 수신 시 Hold 쿨다운이 활성화 중이면 무시
    - 현재 피스를 hold 슬롯에 저장, hold 슬롯 피스(없으면 다음 백 피스)를 스폰
    - 스폰된 피스는 Hold 쿨다운 활성화 → 다음 피스 고정 시 자동 해제

### States and Transitions

| 상태 | 진입 조건 | 탈출 조건 | 동작 |
|------|----------|----------|------|
| **Inactive** | 게임 시작 전 / 스테이지 종료 | 스테이지 시작 | 모든 처리 중단, 입력 무시 |
| **Spawning** | 피스 고정 직후 / 스테이지 시작 | 스폰 성공 또는 충돌 | 새 피스 생성 및 충돌 판정 |
| **Active** | 스폰 성공 | 낙하 불가 도달 | 이동·회전·낙하 처리 |
| **Locking** | 낙하 불가 도달 | lock_timer 만료 또는 하드 드롭 | lock_timer 진행, 이동/회전 성공 시 lock_timer 리셋 |
| **LineClear** | 피스 고정 후 full row 존재 | 클리어 완료 | `clear_row()` 실행, 시그널 발신, 다시 Spawning으로 |
| **GameOver** | 스폰 충돌 발생 | `reset()` 호출 | 모든 처리 중단, `game_over` 시그널 발신 |

### Interactions with Other Systems

| 시스템 | 방향 | 인터페이스 |
|--------|------|-----------|
| **입력 시스템** | 입력 → 코어 | 시그널 수신: `move_left`, `move_right`, `rotate_cw`, `rotate_ccw`, `hard_drop`, `soft_drop_started`, `soft_drop_ended`, `hold_requested` |
| **게임 보드** | 코어 → 보드 | `is_empty()` (이동/회전 충돌 판정), `place_block()` (피스 고정), `get_full_rows()` (클리어 판정), `clear_row()` (라인 삭제) |
| **미션 조건 시스템** | 코어 → 미션 | 시그널 발신: `line_cleared(count: int)`, `piece_locked(type_id: int)`, `hard_drop_used` |
| **게임 HUD** | 코어 → HUD | 시그널 발신: `active_piece_changed`, `hold_changed`, `next_changed`, `line_cleared(count: int)` |
| **씬 관리자** | 코어 → 씬 | 시그널 발신: `game_over` |
| **스킬 시스템** | 스킬 → 코어 | (Alpha) 낙하 속도 오버라이드 수신, Frozen 상태 진입/해제 수신 |

## Formulas

### 낙하 속도 테이블

```
fall_interval (초) — 스테이지별 설정값 (외부 데이터로 관리)
스테이지 1:  1.000s  (여유 있는 시작)
스테이지 2:  0.800s
스테이지 3:  0.650s
스테이지 4:  0.500s
스테이지 5:  0.400s
스테이지 6:  0.300s
스테이지 7:  0.220s
스테이지 8:  0.150s
스테이지 9:  0.100s
스테이지 10: 0.067s  (약 15칸/초, 고숙련자 수준)
```

| 변수 | 타입 | 범위 | 설명 |
|------|------|------|------|
| `fall_interval` | float | 0.067 ~ 1.0 | 1칸 낙하 간격 (초). 낮을수록 빠름 |
| `base_fall_interval` | float | — | 현재 스테이지의 기본 fall_interval |
| `effective_fall_interval` | float | — | 소프트 드롭 시 `base_fall_interval / SDF` |

### 소프트 드롭 속도

```
effective_fall_interval = base_fall_interval / SDF
```

| 변수 | 기본값 | 예시 (스테이지 1) |
|------|--------|-----------------|
| `SDF` | 15.0 | 1.000 / 15 = 0.067s (15칸/초) |

### Ghost Piece 위치

```
ghost_row = active_piece.row
while can_move_down(ghost_row + 1):
    ghost_row += 1
// ghost_row = 하드 드롭 시 고정될 최하단 행
```

### SRS Wall Kick 오프셋 테이블

**J, L, S, T, Z 피스 (CW/CCW 공통)**

| 회전 전→후 | 시도 1 | 시도 2 | 시도 3 | 시도 4 |
|-----------|--------|--------|--------|--------|
| 0 → CW | (−1, 0) | (−1,+1) | (0,−2) | (−1,−2) |
| CW → 180 | (+1, 0) | (+1,−1) | (0,+2) | (+1,+2) |
| 180 → CCW | (+1, 0) | (+1,+1) | (0,−2) | (+1,−2) |
| CCW → 0 | (−1, 0) | (−1,−1) | (0,+2) | (−1,+2) |

**I 피스 (별도 테이블)**

| 회전 전→후 | 시도 1 | 시도 2 | 시도 3 | 시도 4 |
|-----------|--------|--------|--------|--------|
| 0 → CW | (−2, 0) | (+1, 0) | (−2,−1) | (+1,+2) |
| CW → 180 | (−1, 0) | (+2, 0) | (−1,+2) | (+2,−1) |
| 180 → CCW | (+2, 0) | (−1, 0) | (+2,+1) | (−1,−2) |
| CCW → 0 | (+1, 0) | (−2, 0) | (+1,−2) | (−2,+1) |

*(O 피스는 Wall Kick 없음 — 항상 중심 회전)*

> 오프셋 좌표: `(col_offset, row_offset)`. row+ = 아래 방향.
> 모든 오프셋 실패 시 회전 취소 (원래 rotation 상태 복원).

## Edge Cases

| 시나리오 | 예상 동작 | 근거 |
|---------|----------|------|
| 스폰 위치에 블록이 있음 | 즉시 GameOver 상태 전환, `game_over` 시그널 발신 | 표준 테트리스 규칙 |
| Lock Delay 중 이동으로 낙하 가능 공간이 생김 | Active 상태 복귀, lock_timer 리셋 | 피스가 다시 낙하 가능하므로 Locking 해제 |
| Hold 슬롯이 비어있을 때 Hold 요청 | 현재 피스를 Hold에 저장, 다음 백 피스를 즉시 스폰. Hold 쿨다운 활성화 | Hold 슬롯 첫 사용 |
| Hold 쿨다운 중 Hold 요청 | 무시 — 한 피스 주기에 Hold는 1회만 허용 | 무한 Hold 방지 |
| 소프트 드롭 중 하드 드롭 입력 | `hard_drop` 우선 처리 (`soft_drop_ended` 발신 없음) | 명시적 조작 우선 (입력 시스템 GDD 일치) |
| 동시에 4줄이 꽉 참 | 4줄 모두 동시에 `clear_row()` 호출, 위 행 4칸 내려옴 | 게임 보드 GDD 일치. "테트리스!" |
| I 피스 수평 상태에서 회전 시 보드 상단 밖으로 나갈 경우 | SRS Wall Kick 오프셋으로 보정. 모든 오프셋 실패 시 회전 취소 | SRS 규칙 적용 |
| Inactive 상태에서 입력 시그널 수신 | 모든 시그널 무시 | 상태 전환 테이블 준수 |
| 스테이지 클리어 순간 낙하 중인 피스가 있음 | Inactive로 즉시 전환, 진행 중 처리 중단 | 씬 관리자가 Inactive 전환 트리거 |
| `fall_timer` 누적값이 `fall_interval`보다 훨씬 큰 경우 (프레임 드롭) | 1칸만 낙하 후 `fall_timer = 0` 리셋. 한 프레임에 복수 낙하 없음 | 프레임 드롭 시 블록 순간이동 방지 |
| Ghost Piece와 Active Piece가 같은 위치 | Ghost 표시 안 함 (이미 최하단에 위치) | 시각적 혼란 방지 |

## Dependencies

| 시스템 | 방향 | 의존 성격 |
|--------|------|---------|
| **게임 보드** | 테트리스 코어가 의존 | Hard — 충돌 판정, 블록 고정, 라인 클리어 모두 보드 API 필수 |
| **입력 시스템** | 테트리스 코어가 의존 | Hard — 플레이어 입력 없이 피스 조작 불가 |
| **미션 조건 시스템** | 미션이 이 시스템에 의존 | Hard — `line_cleared`, `piece_locked` 시그널이 미션 진행도의 주요 입력 |
| **게임 HUD** | HUD가 이 시스템에 의존 | Hard — Active Piece, Hold, Next, 라인 클리어 정보 없이 HUD 렌더링 불가 |
| **게임 오버 화면** | 게임 오버 화면이 이 시스템에 의존 | Hard — `game_over` 시그널로 화면 전환 트리거 |
| **스킬 시스템** | 스킬이 이 시스템에 의존 (Alpha) | Soft — 낙하 속도 오버라이드, Frozen 상태 제어 |

> 게임 보드 GDD 교차 확인: "테트리스 코어가 이 시스템에 의존 — Hard" ✅
> 입력 시스템 GDD 교차 확인: "테트리스 코어가 이 시스템에 의존 — Hard" ✅

## Tuning Knobs

| 파라미터 | 현재 값 | 안전 범위 | 높이면 | 낮추면 |
|---------|--------|---------|--------|--------|
| `fall_interval` (스테이지별 테이블) | 0.067 ~ 1.0s | 0.033 ~ 2.0s | 게임이 느려져 긴장감 감소 | 너무 빠르면 반응 불가 수준 |
| `LOCK_DELAY` | 0.500s | 0.200 ~ 0.800s | 마지막 배치 조정 시간 증가 | 빠른 고정, 실수 증가, 압박감 상승 |
| `SDF` | 15.0 | 5.0 ~ 40.0 | 소프트 드롭이 하드 드롭에 근접 | 소프트/정상 낙하 차이가 미미해짐 |
| `DAS` | 0.167s | 0.05 ~ 0.3s | 연속 이동이 늦게 시작 | 의도치 않은 연속 이동 증가 |
| `ARR` | 0.033s | 0.0 ~ 0.1s | 연속 이동 속도 느려짐 | 0이면 DAS 후 즉시 벽까지 이동 |

> **소유권 명확화**: `DAS`, `ARR`, `SDF` 는 입력 시스템 GDD 소유. 테트리스 코어는 `base_fall_interval`, `LOCK_DELAY`만 소유.
> **낙하 속도 테이블** 전체는 외부 Resource/JSON으로 관리. 코드에 하드코딩 금지 (Coding Standards 준수).

| Hold 쿨다운 표시 | Hold 슬롯 위 어둡게 처리 | Hold 쿨다운 변경 시 | 쿨다운 활성 중 |

## Acceptance Criteria

**기본 동작**
- [ ] 7종 테트리미노(I, O, T, S, Z, J, L)가 모두 스폰된다
- [ ] 7-Bag 랜덤마이저: 7개가 전부 나오기 전에 같은 피스가 2회 연속 스폰되지 않는다
- [ ] Next 프리뷰에 다음 피스가 올바르게 표시된다

**이동/회전**
- [ ] 좌/우 이동 시 보드 벽 밖으로 나가지 않는다
- [ ] `rotate_cw` 후 `rotate_ccw`를 하면 원래 방향으로 돌아온다
- [ ] I 피스, T 피스의 SRS Wall Kick이 벽 근처에서 정상 동작한다
- [ ] O 피스는 위치 변화 없이 회전만 된다

**낙하/고정**
- [ ] 스테이지 1에서 1초에 1칸 낙하한다 (`fall_interval = 1.0s`)
- [ ] 소프트 드롭 시 낙하 속도가 정상의 15배(SDF 기본값)가 된다
- [ ] 하드 드롭 시 Ghost Piece 위치에 즉시 고정된다
- [ ] Lock Delay(500ms) 동안 이동/회전이 가능하다
- [ ] Lock Delay 중 이동으로 낙하 가능해지면 Active 상태로 복귀한다
- [ ] 프레임 드롭 시에도 한 프레임에 1칸 이상 낙하하지 않는다

**홀드**
- [ ] Hold 요청 시 현재 피스가 홀드 슬롯에 저장되고 새 피스가 스폰된다
- [ ] 같은 피스 주기에 Hold를 2회 요청하면 두 번째는 무시된다
- [ ] 피스 고정 후 Hold 쿨다운이 해제된다

**라인 클리어**
- [ ] 10칸이 채워진 행이 즉시 삭제되고 위 행들이 내려온다
- [ ] 4줄 동시 클리어(테트리스)가 정상 동작한다
- [ ] `line_cleared(count: int)` 시그널이 클리어된 줄 수와 함께 발신된다

**게임 오버**
- [ ] 스폰 위치에 블록이 있으면 `game_over` 시그널이 발신된다

**성능 및 데이터**
- [ ] 모든 테트리스 코어 로직이 프레임당 2ms 이내에 완료된다
- [ ] `fall_interval` 테이블과 `LOCK_DELAY`가 코드에 하드코딩되지 않고 외부 데이터에서 읽힌다
